# Logic Layer

Pure TypeScript business logic ‚Äî NO Svelte/UI/browser dependencies. Enforced by ESLint `no-restricted-imports` + `no-restricted-globals` + pre-commit hook.

## CONVENTIONS

- **SupabaseClient as first param**: All data/auth functions take `supabase: SupabaseClient` as first argument. The singleton client from `$lib/supabase.ts` is never imported here ‚Äî UI layer passes it.
- **Validate before mutate**: `createQSO` calls `validateQSO` before inserting. Validation returns `ValidationResult` with `{ valid, errors }`.
- **Barrel exports**: Each subdirectory has `index.ts` re-exporting public API. Import from `$lib/logic/data`, not `$lib/logic/data/qso`.
- **Co-located tests**: `qso.test.ts` lives next to `qso.ts`. Test fixtures in `tests/fixtures/` are ADIF files.
- **Domain types are const**: `BANDS`, `MODES`, `CONTINENTS` use `as const` for type-safe enums.

## ANTI-PATTERNS

- **NEVER** import svelte, `$app/*`, `$lib/ui/*`, `$lib/supabase`, or routes ‚Äî ESLint will reject
- **NEVER** use browser globals (`window`, `document`, `navigator`, etc.) ‚Äî ESLint will reject
- **NEVER** edit `types/database.ts` ‚Äî auto-generated by `pnpm gen-types`
- **NEVER** import the `supabase` singleton ‚Äî accept it as a parameter instead

## NOTES

- Callsign regex: `/^[A-Z0-9]{1,3}\d[A-Z0-9]{1,4}(?:\/(?:P|M|MM))?$/`
- Grid square: 4 or 6 character Maidenhead locator (`/[A-R]{2}\d{2}(?:[A-X]{2})?/`)
- RST validation differs by mode: phone modes (SSB/FM/AM) allow 2-digit, CW/digital requires 3-digit
- ADIF field mappings in `adif/fields.ts` translate between ADIF tags and DB column names
- `is_eyeball: boolean` on QSO ‚Äî when true, band/freq are optional (face-to-face contacts)
