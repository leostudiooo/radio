## CONVENTIONS

- **Architecture enforced at lint level**: `src/lib/logic/` cannot import from `svelte`, `$app/*`, `$lib/ui/*`, `$lib/supabase`, routes, or use browser globals (`window`, `document`, `localStorage`, `navigator`). ESLint `no-restricted-imports` + `no-restricted-globals` + pre-commit hook all enforce this.
- **Svelte 5 runes required**: All `.svelte` files use `$state`, `$derived`, `$props()`, `$effect`. Old reactive declarations (`$:`) are rejected by compiler. Stores use `.svelte.ts` extension with `$state`/`$derived` — NOT `writable()`/`derived()`.
- **Data functions take SupabaseClient as first param**: Logic layer never imports the singleton `supabase` client. UI layer passes it: `createQSO(supabase, data)`.
- **Co-located tests**: `qso.test.ts` lives next to `qso.ts`, not in a separate directory.
- **Formatting**: Tabs, single quotes, no trailing commas, 100 char print width. Prettier + Tailwind class ordering enforced.
- **Database types auto-generated**: `src/lib/logic/types/database.ts` is gitignored, generated by `pnpm gen-types`. Never edit manually.
- **i18n domain-split**: Translation files per domain (`common.ts`, `qso.ts`, `auth.ts`, etc.) under `src/lib/i18n/{en,zh}/`. Add string → add to both locales.
- **Routes use `onMount`** for data loading (SPA mode, no SSR, no `load` functions).

## ANTI-PATTERNS (THIS PROJECT)

- **NEVER** import Svelte/UI/route code from `src/lib/logic/` — enforced by ESLint + pre-commit hook
- **NEVER** edit `src/lib/logic/types/database.ts` manually — it's auto-generated
- **NEVER** use `writable()`/`derived()` from `svelte/store` — use Svelte 5 runes instead
- **NEVER** use `$:` reactive declarations — use `$derived` or `$effect`
- **NEVER** use `as any` or `@ts-ignore` — fix the type properly
- **DO NOT** create SvelteKit server endpoints (`+server.ts`) — this is an SPA with `ssr=false`
- **DO NOT** add `load()` functions in `+page.ts` — use `onMount` in the component instead

- Ham radio domain types: callsign regex (`/^[A-Z0-9]{1,3}\d[A-Z0-9]{1,4}/`), Maidenhead grid squares, ADIF format, RST signal reports
- Station identity configured in `src/lib/config.ts` (BA4VUN)
- `build/` directory committed to git for direct CF Pages deployment
